# Development Workflow

## Project Overview

Keep in Touch is a full stack personal CRM application.

- `frontend/` contains the Angular frontend.
- `backend/` contains the Spring Boot backend.
- Postgres runs through Docker Compose.
- The frontend talks to the backend through GraphQL.
- Database schema changes are managed with Flyway migrations.
- Frontend GraphQL types and documents are generated with GraphQL Code Generator.

## General Workflow Rules

Before starting work, read:

- `docs/plan.md`
- `docs/schema.md`
- `docs/development.md`
- `README.md`

Work only on the requested phase or task. Do not silently add new frameworks, major dependencies, features, or schema changes. Do not move to the next phase unless explicitly asked.

If a schema, API, or product decision is needed, stop and ask or report the decision point clearly. After work is complete, report files changed, commands run, tests run, and any assumptions made.

## Starting Local Infrastructure

From the repo root:

```bash
docker compose up -d
docker compose ps
```

`docker compose down` stops containers but keeps the named volume. `docker compose down -v` deletes local database data and should only be used when intentionally resetting the database.

## Backend Workflow

```bash
cd backend
SPRING_PROFILES_ACTIVE=local ./mvnw spring-boot:run
./mvnw test
```

Flyway migrations run on backend startup. Schema changes should be made through new Flyway migration files, not Hibernate auto-DDL.

GraphQL schema changes live in:

```text
backend/src/main/resources/graphql/schema.graphqls
```

## Frontend Workflow

```bash
cd frontend
npm install
npm start
npm run codegen
npm run lint
npm run build
```

`npm start` runs the Angular dev server with the configured proxy. Angular tests may require Chrome or ChromeHeadless depending on the local setup. Do not install Chrome or Chromium automatically unless explicitly approved.

## GraphQL Codegen Workflow

Frontend GraphQL operations live in:

```text
frontend/src/app/core/graphql/crm.graphql
```

Generated types and typed documents live in:

```text
frontend/src/app/core/graphql/generated.ts
```

Codegen config lives in:

```text
frontend/codegen.ts
```

When GraphQL operations or the backend schema change, run:

```bash
cd frontend
npm run codegen
npm run lint
npm run build
```

`generated.ts` is generated but intentionally committed. Frontend app code should use generated types and documents instead of manually duplicating GraphQL response or input types.

## Before Committing

From the repo root or appropriate folders, run:

```bash
docker compose ps
cd frontend && npm run codegen && npm run lint && npm run build
cd ../backend && ./mvnw test
```

Do not run:

```bash
npm audit fix --force
```

unless explicitly approved.

## Agent-Specific Instructions

AI and LLM coding agents should follow these rules:

- Do not make broad refactors unless asked.
- Do not "improve" UI, schema, or architecture outside the requested task.
- Prefer small, reviewable changes.
- Keep generated files generated by the proper command.
- If editing GraphQL operations, run codegen afterward.
- If editing backend schema, check whether frontend codegen needs to be rerun.
- If adding dependencies, explain why they are necessary.
- If a command fails, report the exact command and error instead of working around it silently.

## Constraints

- Do not change backend behavior unless explicitly asked.
- Do not redesign UI.
- Do not move to Phase 6 without explicit approval.
- Do not add interactions, follow-ups, dashboard, auth, tags, recurring follow-ups, notifications, AI features, or future-phase work unless explicitly requested.
- Do not run `npm audit fix --force`.
- Do not install Chrome or Chromium unless explicitly approved.
